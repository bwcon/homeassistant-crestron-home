# Crestron Home Integration for Home Assistant

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-41BDF5.svg)](https://github.com/hacs/integration)
![Home Assistant](https://img.shields.io/badge/Home%20Assistant-Custom%20Component-blue)
![Python](https://img.shields.io/badge/Python-3.10%2B-blue)

An elegant, robust, and local-polling custom integration for Home Assistant that bridges **Crestron Home® OS** (R-Series System Processors) with Home Assistant. It exposes your Crestron Home rooms and devices as native Home Assistant entities with high-performance 2-way polling feedback.

---

## Features

*   **Zero Controller Deployment**: Communicates entirely via Crestron's official local REST Web API (CWS). No custom SIMPL, SIMPL+, or SIMPL# programs need to be loaded onto the processor.
*   **Dynamic Room & Device Discovery**: Automatically parses your Crestron Home layout and maps entities to their respective rooms/areas in Home Assistant.
*   **Lights & Dimmers**: Exposes dimmer and switch-type lights with seamless 16-bit to 8-bit brightness scaling (`0-65535` $\leftrightarrow$ `0-255`). Optimistic updates ensure near-instant UI reactivity.
*   **Window Shades**: Exposes shades with position scaling (`0-65535` $\leftrightarrow$ `0-100% open`).
*   **HVAC / Thermostats**: Full climate control including heating/cooling setpoints, system modes (Off, Heat, Cool, Auto), and fan modes (Auto, On). Handles dual setpoints in Auto mode, and converts Crestron's `DeciFahrenheit` (e.g. `720` for `72.0°F`) and `CelsiusHalfDegrees` (e.g. `235` for `23.5°C`) automatically.
*   **Scenes**: Exposes pre-configured Crestron lighting/shade scenes as native Home Assistant scenes.
*   **Door Locks**: Supports locking, unlocking, and real-time status reporting.
*   **Binary Sensors**: Auto-discovers occupancy/motion sensors and door/window contact sensors.
*   **Diagnostic Sensors**: Exposes light level (illuminance in lux) and battery levels.
*   **Parallel Polling Coordinator**: Utilizes `asyncio.gather` to fetch all device, lock, and scene states in parallel every 15 seconds, minimizing CPU overhead and network requests.

## Installation

### Installation via HACS (Recommended)

1. Make sure **HACS** is installed and configured in your Home Assistant instance.
2. Navigate to **HACS** in the Home Assistant sidebar.
3. Click the **three dots (⋮)** in the top-right corner of the HACS panel and select **Custom repositories**.
4. In the **Repository** field, paste the URL of this repository: `https://github.com/bwcon/homeassistant-crestron-home`
5. In the **Category** dropdown, select **Integration**.
6. Click **Add**.
7. Find the newly added **Crestron Home Integration for Home Assistant** in HACS, click on it, and click **Download** in the bottom right.
8. **Restart** Home Assistant to load the custom component.

### Manual Installation

1.  Download the repository.
2.  Copy the `custom_components/crestron_home/` folder to your Home Assistant configuration directory under the `custom_components/` directory (create it if it does not exist).
3.  Restart Home Assistant.

## Configuration

To add the integration, configure it through the Home Assistant frontend:

1.  Navigate to **Settings > Devices & Services**.
2.  Click **Add Integration** in the bottom-right corner.
3.  Search for and select **Crestron Home**.
4.  Fill in the configuration details:
    *   **IP Address or Hostname**: The local IP address of your Crestron system processor.
    *   **Connection Protocol**: Select `https` (recommended) or `http`.
    *   **API Port**: Default is `443` (for HTTPS) or `80` (for HTTP).
    *   **Web API Token**: Enter the authorization token generated from the Crestron Home setup (see instructions below).
    *   **Verify SSL Certificate**: Check this only if you have a valid, trusted SSL certificate loaded on your processor. Leave unchecked (default) to bypass verification for self-signed certificates.
5.  Click **Submit** to finalize the integration. All entities will be added under a unified "Crestron Home Processor" device.

### How to Retrieve the Web API Token

To connect the integration, you need a token generated by the Crestron Home setup application:

1.  Open the **Crestron Home Setup App** (for installers).
2.  Go to **Settings > Installer Settings > System Control Options > Web API Settings**.
3.  Ensure the Web API is enabled.
4.  Select **Update Token** to generate a new token string.
5.  Copy this token and paste it into the Home Assistant configuration setup.

*Note: If you perform a firmware upgrade on your Crestron processor, you may need to regenerate and update the token.*

## Local Verification & Mock Testing

The repository includes a python-based mock server and test script to verify connection logic offline:

1.  Navigate to the `scratch/` directory (or where you put the mock files).
2.  Run the mock server to simulate a processor:
    ```bash
    python mock_crestron_server.py
    ```
3.  In another terminal, run the test script:
    ```bash
    python test_api_client.py
    ```
This runs a suite of tests ensuring that the async client wrapper authenticates, fetches data, maps levels, and controls devices against the CWS endpoints.
